# Supabase SQL Schema for Zarq Workspace KYC

This SQL schema matches the current app model in `src/types.ts` and the Supabase integration in `src/lib/supabase.ts`.

Use this in Supabase SQL Editor or your database migration tool.

## Recommended table schema

```sql
-- Create enum types for strong data integrity
CREATE TYPE IF NOT EXISTS subscription_plan AS ENUM (
  'weekly-desk',
  'weekly-pc',
  'monthly-desk',
  'monthly-pc'
);

CREATE TYPE IF NOT EXISTS customer_kyc_status AS ENUM (
  'pending',
  'approved',
  'rejected'
);

-- Create the customers table
CREATE TABLE IF NOT EXISTS public.customers (
  id text PRIMARY KEY,
  full_name text NOT NULL,
  phone_number text NOT NULL,
  national_id text NOT NULL,
  nationality text NOT NULL,
  estate_area text NOT NULL,
  subscription_plan subscription_plan NOT NULL,
  nature_of_work text NOT NULL,
  nature_of_work_other text,
  company_name text,
  description_of_activities text NOT NULL,
  emergency_contact jsonb,
  terms_accepted boolean NOT NULL DEFAULT false,
  status customer_kyc_status NOT NULL DEFAULT 'pending',
  registration_date timestamptz NOT NULL DEFAULT timezone('utc'::text, now()),
  document_url text,
  document_name text,
  created_at timestamptz NOT NULL DEFAULT timezone('utc'::text, now())
);

-- Optional index for faster status filtering in the admin dashboard
CREATE INDEX IF NOT EXISTS idx_customers_status ON public.customers (status);

-- Optional index for fetch order and quick lookups by registration date
CREATE INDEX IF NOT EXISTS idx_customers_registration_date ON public.customers (registration_date DESC);
```

## Notes

- The app expects the table name to be `customers`.
- `id` is generated by the app as a string like `ZARQ-2026-0001`.
- `emergency_contact` is stored as JSON and may include `fullName`, `relationship`, and `phoneNumber`.
- `document_url` is reserved for any file URL or placeholder text, while `document_name` stores the uploaded file name.
- `terms_accepted` is required by KYC flow and should be `true` for valid submissions.

## Supabase setup tip

If you want Supabase to host uploaded document files later, use a Storage bucket and save the public/private file URL in `document_url`.
